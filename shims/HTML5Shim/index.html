HTML5 Semantic Tags

The (w3c) HTML5 semantic elements provide a means of accessing new features, i.e. Canvas, Video, Audio, whilst also providing a means of standrdizing the naming of common page layouts, such as header, footer, and nav.


However the style of these elements does not appear correctly on IE<=8. So this page uses the shim html5shiv by remy sharp to correctly display the <header> and <footer> tags.

While using this shim you you should specify if they are inline or block level. If not all the new elements will be of type inline. If you plan to support Firefox 3.6 you should also add this CSS styles!
